# Прогнозирование цен на недвижимость в Москве

Проект по предсказанию стоимости квартир на основе данных о площади, расположении,
количестве комнат и других характеристиках. Портфолио-проект для практики полного
цикла работы с данными: от очистки до обучения модели.

## Описание задачи

Цель — построить модель, предсказывающую цену квартиры по её характеристикам
(площадь, количество комнат, этаж, расстояние до метро, регион и др.).
Задача регрессии, целевая переменная — `Price`.

## Данные

- Источник: датасет объявлений о продаже квартир (Москва и Московская область) взятый с kaggle 
- Объём: изначально ~[22676] записей, после очистки — [18091] записей
- Основные признаки: площадь (общая/жилая/кухня), количество комнат, этаж,
  этажность дома, расстояние до метро, станция метро, регион


## EDA — ключевые находки

- **Целевая переменная (Price)** сильно скошена (skewness ≈ 7.2), присутствуют
  экстремальные выбросы. Логарифмирование (`log1p`) существенно нормализует
  распределение.
- **Нелинейная зависимость цены от площади** — цена растёт непропорционально
  быстрее для больших квартир. Это мотивирует выбор моделей на основе деревьев
  решений (CatBoost, Random Forest) как основных.
- **Расстояние до метро** обратно связано с ценой, как и ожидалось.
- **Станция метро** — признак высокой кардинальности (500+ уникальных значений),
  требует аккуратного кодирования (target encoding или прямая передача в CatBoost).
- **Мультиколлинеарность**: `Area` и `Living area` сильно коррелируют (r = 0.89).
  Решение — заменить `Living area` на производный признак `living_area_ratio`
  (доля жилой площади от общей).

## Очистка данных

Применены два типа фильтров:

1. **Логические противоречия** (безусловное удаление):
   - `Living area > Area`
   - `Living area + Kitchen area > Area`
   - `Floor > Number of floors`

2. **Статистические выбросы**:
   - Квартиры с `rooms == 0` (студии) — оставлены как валидный сегмент,
     но выбросы по площади внутри группы удалены через IQR-метод
   - Перцентильная фильтрация (1–99%) по `Price`, `Area`, `Living area`,
     `Kitchen area`

## Модель

- **Baseline**: линейная регрессия на логарифмированных `Price` и `Area`
  и остальных числовых/закодированных признаках
- **Основная модель**: CatBoost — выбран из-за нативной поддержки
  категориальных признаков высокой кардинальности (станция метро, 500+
  значений) и устойчивости к обнаруженной нелинейной зависимости цены
  от площади
- **Валидация**: 5-fold кросс-валидация для надёжной оценки качества
  перед подбором гиперпараметров
- **Подбор гиперпараметров**: Grid Search (полная сетка, 81 комбинация ×
  3 фолда) по `iterations`, `learning_rate`, `depth`, `l2_leaf_reg`
- **Метрики**: MAE, RMSE, MAPE (в рублях/процентах, после обратного
  преобразования из логарифма предсказаний)

## Результаты

| Модель                        | MAE (руб)  | RMSE (руб) | MAPE (%)      |
|--------------------------------|-----------|------------|---------------|
| Linear Regression (baseline)   | 10,105,120 | 32,887,030 | 22.62         |
| CatBoost (параметры по умолчанию) | 6,260,836  | 19,431,130 | 13.47         |
| CatBoost (5-fold CV)           | —         | —          | 13.59 ± 0.31  |
| CatBoost (после Grid Search)   | —         | —          | **12.33**     |

**Лучшие гиперпараметры**: `depth=6`, `learning_rate=0.1`, `l2_leaf_reg=3`,
`iterations=700`

**Проверка на переобучение**: Train MAPE = 10.31%, Test MAPE = 12.33% —
разница небольшая, признаков серьёзного переобучения нет.

**Наиболее значимые признаки** (CatBoost feature importance):
1. `Area` — 47.0%
2. `Apartment type` — 11.1%
3. `Renovation` — 9.9%
4. `Region` — 9.0%
5. `Metro station` — 6.3%

CatBoost с подобранными гиперпараметрами улучшил MAPE почти вдвое
относительно linear regression baseline (22.62% → 12.33%) и примерно на
1.3 п.п. относительно параметров по умолчанию (13.47% → 12.33%),
что подтверждает как ценность выбора модели под структуру данных, так и
пользу от тюнинга гиперпараметров.
